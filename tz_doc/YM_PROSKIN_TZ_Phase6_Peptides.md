## Блок A — Сбор фактов + живой гейт
1. Собрать факты по пептидам через Exa + PubMed/NCBI живьём (IPv4-force + UA + throttle; P009
   verify-at-write; P014 — кэш пишет только живой фетчер).
2. `[GUARD]` **Источник↔тезис:** источник факта должен подтверждать ИМЕННО этот тезис, а не быть
   «по теме рядом». Свежий пример риска: в vitamin_c тезис про мРНК коллагена оказался привязан к
   статье Pinnell об *абсорбции* (PMID 11207686) — так нельзя. Сверять содержание абстракта с тезисом.
3. `[GUARD]` **Пептиды — поле с маркетинговым хайпом и тонкой доказательной базой** («ботокс в
   баночке»). Ждать ВЫСОКОГО процента брака и много WEAK; браковать и карантинить, **не добивать до
   порога мусором** (P007). Реальный процент брака сообщить; **0% брака = красный флаг** (P014).
4. Целевые сущности (классы и примеры; контент — только из гейта): сигнальные (palmitoyl
   pentapeptide-4 / Matrixyl, palmitoyl tripeptide), несущие (GHK-Cu / медный трипептид),
   ингибиторы нейротрансмиттеров (acetyl hexapeptide-8 / Argireline), ингибиторы ферментов.
   Цель — выйти на **≥8 чистых фактов** (P011 deck-ready).

## Блок B — Онтология / граф
1. Добавить сущности-пептиды + факты + связи через A07; provenance обязателен (P009).
2. `validate_graph.py` зелёный; новые факты сверены источник↔тезис (Блок A2).

## Блок C — Spec колоды
1. Контент-ведомые макеты (P018 слайд↔факт); слайд безопасности — из фактов (пептиды обычно хорошо
   переносятся; topic-aware safety через `safety_config.json`, **без выдуманных противопоказаний**).
2. Дисклеймеры «Только для специалистов»; title ≤50, subtitle ≤100; цвет/шрифт не в spec.
3. Валидировать по `schemas/slide-spec.json`.

## Блок D — Рендер + визуалы
1. Dual render PDF+PPTX из единого spec.
2. `[GUARD]` **Механизм-диаграммы НЕ генерировать** в этой фазе — подписанные плейсхолдеры
   (метод схем перерабатывается отдельно). Не тиражировать текущий SVG-подход на пептиды.
3. Молекулы: опционально. Крупные пептиды в 2D через RDKit дают мусорный «развал» — генерировать
   ТОЛЬКО если читаемо; иначе sequence/INCI-обозначение, без насильной кривой структуры. Любой
   ассет → provenance + license-гейт (P019).

## Блок E — QA
`qa_deck.py`: слайд↔факт (P018), topic-aware safety, Zero Black, Arimo, license/provenance,
плейсхолдеры явно помечены, спот-чек источник↔тезис. Все существующие гейты держатся.
