## 0. ЗАЧЕМ ЭТА ФАЗА (читать первым)

Прошлая ошибка проекта: агент сделал **6 деков по 7–8 слайдов** вместо **2 по 13–20**, и заказчице нечего показать. Эта фаза закрывает это: производит **2 закреплённых каркаса (Ретиноиды 16 слайдов, Постакне 18 слайдов)** и рендерит их **пустыми** (placeholder-плашки + бренд-хром, без контента) в PDF — чтобы заказчица увидела структуру и утвердила ДО наполнения.

[GUARD] Это НЕ универсальный движок на 12 интентов (DEC-020 предостерёг). Только то, что нужно для 2 пилотных дек.

## 2. LAYOUT-СИСТЕМА: 4 КЛАССА (Блок B)

Принят каркас из кросс-ревью (GPT), с обязательной правкой под анти-фабрикацию. Layout'ы группируются в 4 класса по функции:

### Класс A — EXPERT AUTHORITY (ведущие, демонстрируют мышление)
- **A1 Classification** — тема разложена на N категорий (карточки/дерево). Пример: Постакне → пигментация / сосудистый / рубцы / текстура.
- **A2 Framework** — авторская система оценки по осям. Пример: ретиноид по 4 параметрам (эффективность / раздражение / стабильность / доказательность).
- **A3 Decision Map** — концептуальная карта решений (НЕ клинический протокол назначений; см. GUARD ниже).
- **A4 Matrix** — 2D-сетка (активы × свойства).

### Класс B — KNOWLEDGE DELIVERY (объясняют механику)
- **B1 Split** — placeholder слева (схема/молекула) + текст справа.
- **B2 Timeline/Process** — пошаговая шкала (конверсия, протокол).
- **B3 Anatomy** — placeholder схемы кожи/слоёв + подписи.

### Класс C — EVIDENCE (поддерживают, не ведут)
- **C1 Comparison** — таблица/2 колонки «А vs Б».
- **C2 Evidence Summary** — сводка фактов с grade-уровнем.
- **C3 Research Chart** — график данных. [GUARD] не больше 1–2 на деку.

### Класс D — TRUST / FRAME
- **D1 Title** — обложка.
- **D2 Section** — разделитель части.
- **D3 Closing** — итоговая карта + badge-лого.
- **D4 Before/After** — [GUARD] осторожно, опционально; реальные пациенты = этический риск, по умолчанию НЕ включать в пилот.

### [GUARD] КРИТИЧНО — анти-фабрикация во фреймворках (моя правка поверх GPT)
GPT сказал «неважно что внутри фреймворка». **Для этого проекта — наоборот, важно.**
- Каждая ячейка Classification / Framework / Matrix / Decision Map, делающая фактическое утверждение (число, эффект, механизм), ОБЯЗАНА трассироваться к `fact_id` (P018) ИЛИ быть явно помечена как **авторская организующая рамка** (структура мышления), а не факт.
- Различение видимо зрителю: фактические ячейки несут grade-маркер (A/B/C) или caption-источник; рамочные — нейтральны, без претензии на доказанность.
- [GUARD] Фреймворк не имеет права превращать C-grade/самооценку в выглядящий-как-аксиома пункт. Grade-честность из 8.7b распространяется на фреймворки.
