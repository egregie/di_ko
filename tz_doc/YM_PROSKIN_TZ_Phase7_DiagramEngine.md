# YM PROSKIN — Phase 7: Diagram Engine v2

Версия: 1.0 · Дата: 2026-06-10 · Предыдущий: Phase 6 (принят, тег `phase-6-peptides`).
Цель: убрать дефекты диаграмм **структурно**, а не патчами. Принцип: **профилактика > детекция** —
геометрия вычисляется из шаблона и токенов (наложение/смещение невозможны по построению),
детерминированный чекер ловит остаток, текст живёт в слайде, а не в иллюстрации.

Контекст: ручная починка SVG = whack-a-mole (чинишь одно — ломается другое). Решение — не «учить
модель рисовать», а **принуждение геометрии**.

> Свежей сессии: прочитай `00_PROJECT_STATE/` и `AGENTS.md` (P001–P019), потом исполняй.

`[GUARD]` ВНЕ scope этой фазы (явный запрет):
- **Обучение/файнтюн модели** — не делать. Antigravity — агентная IDE, не платформа обучения.
  «Критик» = детерминированный геометрический чек (+ опц. промпт-агент), НЕ обученная сеть.
- **Ночная автономная сессия без надзора** — не запускать (против правила audit-after-agent).
- **Скрейп фигур из журналов/атласов** — копирайт-ловушка. Правила вёрстки прописываем сами.

Легенда: `[GUARD]` · `[ВЕРИФ user]`.

## Блок 0 — Структура и governance
1. Расширять `04_design_system/` — НЕ плодить параллельную ветку. Формализовать существующий
   `diagram_layout_checklist.md` в машинные токены.
2. `P020 Diagram-engine`: (а) диаграмма несёт только короткие лейблы (≤4 слов, ≤6 лейблов / ≤3
   ключевых тезиса); вся проза — в теле слайда, не в иллюстрации; (б) геометрия диаграммы
   вычисляется шаблоном/токенами, не расставляется вручную; (в) ни одна диаграмма не сохраняется,
   не пройдя детерминированный чек. → в `AGENTS.md`.
3. `DEC-016`: переход на параметрические шаблоны вместо free-form SVG.

## Блок A — Стандарт инфографики (данные, не проза)
`04_design_system/diagram_tokens.json` с конкретными числами:
- safe-zone padding (≥24px со всех сторон), мин. зазор между элементами, line-spacing колонки подписей;
- лимит плотности: макс. лейблов / длина лейбла; правило «текст сверх лимита → в тело слайда»;
- размеры шрифта (Arimo), палитра (существующие токены), `moleculeStroke`.
Геометрия выводится из этих токенов («формула, не на глаз»).

## Блок B — Параметрические шаблоны (ядро)
Малый набор генераторов (Python→SVG), каждый ВЫЧИСЛЯЕТ все координаты из токенов Блока A →
центрирование/отступы/непересечение гарантированы по построению:
1. `process_flow` — горизонтальный поток N-боксов со стрелками (ceramide, collagen, потоковые схемы).
   Вход: `[{label, sublabel}]`; авто-разнос, центрирование текста, бокс по размеру текста+паддинг.
2. `layered_anatomy` — M слоёв-баров + правая колонка стадий с РАВНЫМ вертикальным ритмом (слои кожи).
   Вход: `layers[{name, glyph}]`, `stages[{title, sub}]`; глифы центрированы в барах.
3. `xy_curve` — одно/двухпанельный x/y-график с маркерами пика/порога/зон (абсорбция pH и конц.).
   Вход: оси, тики, форма кривой, callouts с якорями, **авто-смещаемыми от кривой** (это убрало бы
   налезание callout на линию).
`[GUARD]` Каждый шаблон: принимает только короткие лейблы (длинные — отсекать/выносить); эмитит
брендовый SVG (палитра, Arimo, Zero Black, moleculeStroke); сохраняет привязку контента к фактам
(diagram↔факт остаётся в силе).
Сложные бесшаблонные схемы (RAR/RXR) — пока вручную, но обязаны пройти чек (Блок C); при
повторяемости — кандидат на 4-й шаблон (follow-up).

## Блок C — Цикл генерация→чек→перегенерация (бэкстоп)
1. `qa_svg_bounds.js` (расширен в 4.5c: край + наложения текст↔графика, блок↔блок/фон, текст-вне-бокса)
   — детерминированный критик. Любая диаграмма (шаблонная ИЛИ ручная) обязана пройти его ДО
   попадания в деку; FAIL → перегенерация/правка, не сохранять.
2. Опционально и ВТОРИЧНО: промпт-агент-ревизор на «воздух/читаемость» ПОСЛЕ детерминированного
   прохода. `[GUARD]` Это промпт, не обучение; надёжность несёт детерминированная часть.

## Блок D — Миграция существующих схем (тут whack-a-mole закрывается по-настоящему)
1. Перегенерировать 7 схем через шаблоны, где подходят: `skin_layers`→layered_anatomy;
   `ceramide`/`collagen`→process_flow; `absorption`→xy_curve; `melanosome`/`desmolysis`→process_flow
   или bespoke; `rar_rxr`→bespoke+чек. Это структурно устраняет остаточные дефекты 4.5c (а не патчит).
2. `[GUARD]` Шаблоны чинят ГЕОМЕТРИЮ. Корректность СОДЕРЖАНИЯ (пик vs плато, diagram↔факт) — это
   привязка к фактам + доменное ревью пользователя, шаблоны её не заменяют.
3. Перерендерить затронутые деки (4 темы + peptides-плейсхолдеры по мере готовности); прогнать цикл-чек; PASS.
