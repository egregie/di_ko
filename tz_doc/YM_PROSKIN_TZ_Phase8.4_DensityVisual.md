# ТЗ: Phase 8.4 — Плотность эталона: расширение графа + переработка визуального слоя

**Дата:** 2026-06-17
**Тег:** `phase-8.4-density-visual`
**Предусловие:** 8.3 done (деки наполнены, но РЕДКО ~50 слов/слайд). Граф 35/55/55, grade A:21/B:3/C:31.
**Корневая проблема:** слайды пустые НЕ из-за дизайна, а из-за бедности графа. Эталонные деки (NotebookLM, нравятся заказчице) несут ~120–180 слов/слайд из ПОЛНОГО экспертного знания; наши — ~50 из 17 пригодных фактов. Решение (выбрано заказчиком): **расширить граф верифицированными фактами**, плотность берётся из них, НЕ из «налитого текста».
**Метод:** мелкие шаги (Карпатый), коммит перед стартом и на каждом блоке, self-anneal в источнике.

## БЛОК A — РАСШИРЕНИЕ ГРАФА (ядро; плотность отсюда)

### A1. Дефицит-анализ по слайдам
Для каждого контентного слайда обеих дек посчитать: сколько фактов сейчас vs сколько нужно для 6–10 пунктов. Вывести таблицу слайд → текущие fact_id → дефицит N фактов → тема для добора.

### A2. Live-сбор недостающих фактов (как 8.7a)
Темы добора (примерный пул, по дефициту A1):
- **Пигментация:** транексамовая кислота, койевая, арбутин, резорцин, цистеамин, тиоктовая; механизмы (ингибиция тирозиназы vs блок переноса меланосом); IPL/BBL по PIH.
- **Рубцы:** TCA CROSS детали, эрбиевый лазер, ND:YAG, филлеры ГА при атрофии, комбинированные протоколы, шкала Goodman-Baron как факт.
- **PIE (сосудистое):** PDL/сосудистые лазеры, азелаин при эритеме, центелла, ниацинамид при PIE.
- **Ретиноиды:** концентрации, сравнение раздражения, инкапсулированные формы, частота применения по данным.
- **Текстура:** комбинации кислот, полигидроксикислоты, профили переносимости.

Пайплайн: кандидаты → `verify_gate.py --force-live` (grounded judge + abstract) → SUPPORTED/WEAK → entity/fact + relationships → `build_index.py` → `validate_graph.py`.
[GUARD] Историч. rejection 25–33% → собирать с запасом (~1.4× от нужного). 0% брака = красный флаг (P014).
[GUARD] Каждый факт — реальный PMID, живой абстракт, дословная цитата. НЕ переписывать statement под источник (P009).
[ВЕРИФ] raw verdict + цитата по КАЖДОМУ кандидату (прошёл/отбракован).

### A3. Цель добора
Довести постакне-домены и ретиноиды до плотности, где каждый контентный слайд опирается на ≥6 фактов. Ожидаемо +30–50 новых фактов. Grade честный (derive_grade из 8.7b), C-факты остаются C.
[ВЕРИФ] Финал: новый счёт графа, распределение grade, validate_graph PASS (сырой вывод).

## БЛОК B — ВИЗУАЛ: ФОНОВЫЕ МОЛЕКУЛЫ (замечание 5 — критично)

### B1. Молекулы за текстом, РАЗНЫЕ позиции
Сгенерированные SVG-молекулы (которые есть, но НЕ вставлены) — на задний фон контентных слайдов, ПОД текстом (z-order ниже, opacity 8–15%, читаемость текста — приоритет).
[GUARD] КАТЕГОРИЧЕСКИ: НЕ одна позиция на всех слайдах (текущий дефект — штамп справа). Варьировать по принципам композиции:
- позиция по **правилу третей** (молекула в одном из 4 узлов пересечения третей, разный узел на разных слайдах);
- **поворот** 0–360° варьировать;
- **масштаб** варьировать (на одних крупно у края, на других мельче в углу);
- **асимметричный баланс**: если текст слева — молекула смещена вправо-низ, и наоборот (уравновешивать пустоту, а не дублировать заполненную зону).
- использовать РАЗНЫЕ молекулы (ретинол, азелаиновая, ниацинамид, аскорбиновая) под тему слайда, не одну.
[ВЕРИФ] Показать 6+ разных слайдов — позиции/повороты/молекулы должны явно различаться. Скриншоты.

### B2. Принципы (агент применяет САМ, знание есть)
Симметрия/асимметрия, правило третей, визуальный вес, гармония. Молекула уравновешивает композицию, заполняет «воздух» эстетично, не налезает на текст. Это не требует вопросов заказчику — это базовая теория композиции.

## БЛОК C — ВИЗУАЛ: ЛОГОТИП (замечания 2, 3)

### C1. Цвет лого ПО ДОМИНАНТЕ ФОНА (замечание 2 — жёстко)
[GUARD] Контрастная логика, определять цвет фона слайда:
- фон ПРЕОБЛАДАЕТ ЗЕЛЁНЫЙ/сейдж → лого **РОЗОВОЕ** (тинт Soft Beige #E6D2CA).
- фон ПРЕОБЛАДАЕТ РОЗОВЫЙ/беж (bgAlt #E6D2CA, секции) → лого **ЗЕЛЁНОЕ** (Deep Herbal #6D8470 / Pale Sage).
- НЕ нейтрально-серый на всех (текущий дефект).
Определять доминанту программно (какого токена больше площади на слайде) или по типу layout (секции = розовый фон → зелёное лого; контентные = светлый → по акцентам).

### C2. Позиция лого — ПУЛ, не прижато к краю (замечание 3)
[GUARD] НЕ прижимать к краю. Отступ от края ≥5% canvas. Пул позиций, варьировать по типу слайда:
- центр (для секций/закрытия);
- нижний правый с отступом (контентные);
- верхний левый крупно (титул);
- можно фоновая крупная молекула-лого по центру с большим opacity на разделителях.
[ВЕРИФ] Лого на разных слайдах — разные позиции, нигде не прижато.

## БЛОК D — ПЕРЕДЕЛКА СЛАБЫХ СЛАЙДОВ (замечания 1, 4, 6)

### D1. Большие пустые 4-блока (замечание 1)

### D3. Мини-SVG крупнее + у текста (замечания 6, 6.1)
Мини-иконки увеличить (сейчас крохотные). Ставить вплотную к тексту как маркеры-категории (как эталон: иконка + заголовок категории + описание в строку). Размер соразмерен заголовку строки, не точка.

## БЛОК E — НАПОЛНЕНИЕ ПЛОТНОСТЬЮ (замечание 7 — итог)

### E1. Перегенерация контента из расширенного графа
После Блока A `gen_content.py` тянет БОЛЬШЕ фактов на слайд (6–10). Каждый — fact_id + grade-чип (трассируемость by construction сохраняется). Цель 100–150 слов/контентный слайд.
[GUARD] Плотность из фактов, не из воды. Каждый фактический пункт → fact_id. Авторские рамки (категории, оси) — нейтральны.

### E2. Компоновка по эталону
Брать у эталона ПАТТЕРН (не контент): многоколоночные списки активов с описаниями, иконка+категория+текст, картинка делит слайд 50/50 с текстом. Применить к нашим layoutّам.
[GUARD] C-grade факты — с рамкой уровня (обзор/неконтролируемое), не как аксиома (правило 8.3 сохраняется).

## DEFINITION OF DONE
- [ ] Коммит перед стартом.
- [ ] A: дефицит-анализ; +30–50 фактов через live-гейт (raw verdict+цитата каждый); каждый контентный слайд ≥6 фактов; validate_graph PASS; новый счёт показан.
- [ ] B: фоновые молекулы вставлены, РАЗНЫЕ позиции/повороты/масштаб/молекулы по правилу третей и асимметрии; 6+ слайдов-доказательств различия.
- [ ] C1: цвет лого по доминанте фона (зелёный фон→розовое лого и наоборот); C2: лого не прижато (≥5%), пул позиций.
- [ ] D: 4-блоки наполнены/ужаты; секции-разделители переделаны (заголовок смещён, новая SVG, контент-якорь); мини-SVG крупнее и у текста.
- [ ] E: плотность 100–150 слов/контентный слайд из фактов; компоновка по эталону; C-факты с рамкой.
- [ ] F: обе деки → PDF; validate_content (+density) PASS сырой вывод; 8+ скриншотов; validate_graph PASS; negative-control.
- [ ] Коммит на каждом блоке; финал коммит+тег `phase-8.4-density-visual`.

## КРАСНЫЕ ФЛАГИ (отклонить если)
- Плотность добита водой/выдуманными фактами без fact_id (P009).
- 0% rejection на доборе фактов (P014).
- Фоновые молекулы в ОДНОМ месте на всех слайдах (главный дефект, который чиним).
- Лого нейтрального цвета / прижато к краю.
- Молекула/мини-SVG налезает на текст, ломает читаемость.
- C-факт как аксиома без рамки уровня.
- Скриншоты на недоступных локальных путях.
- «PASS» без сырого вывода и без скриншотов реального различия молекул.
