## Блок 0 — Процессная гигиена (правила в `AGENTS.md`)
1. `P016 Deltas-честность:` `deltas_vs_plan: None` допустимо ТОЛЬКО если исполнение точно совпало
   с планом. Любая незапланированная правка (удаление/переименование файлов, чистка дубликатов,
   обходные шаги) ОБЯЗАНА быть в `deltas_vs_plan`. (Расширяет P013.) Пример нарушения: в 4.2
   удаление `pilot_*`-дубликатов не попало в deltas.
2. `P017 Confidence-семантика:` поле `confidence` нельзя проставлять шаблонно (0.95 всем).
   - Сначала проверить, потребляется ли `confidence` где-либо (retriever/render/QA).
   - Если ДА → вывести по правилу из `evidence_level`+`audit_verdict` (напр. A+SUPPORTED=0.95,
     B+SUPPORTED=0.85, WEAK=0.70) и единоразово пересчитать существующие факты скриптом.
   - Если НЕТ → пометить поле в `schemas/fact.json` как `reserved/unused` и перестать заполнять
     ложной точностью. `DEC-014` зафиксировать выбранный вариант.

Acceptance: P016/P017 в `AGENTS.md`; `confidence` либо выведен по правилу, либо помечен reserved;
`DEC-014` внесён.

## Блок A — Планы и spec колод (по теме)
Исходный материал (8 чистых фактов на тему, тянуть ТОЛЬКО через `retriever.py`):
- **vitamin_c**: fact_0021,0022,0028,0029,0030,0044,0045,0046
- **niacinamide**: fact_0024,0025,0034,0035,0036,0038,0039,0047
- **exfoliants**: fact_0017,0018,0019,0020,0041,0048,0049,0050

Для каждой темы — deck-plan ~7–9 слайдов:
1. `[GUARD]` **Контент-ведомость:** макет выбирается под факты, а НЕ «показать все 10». Берём
   только подходящие layouts. Пустые слайды ради витрины — запрещены.
2. Структура: cover → 4–6 контентных слайдов (механизм/применение/сравнение/данные) →
   **слайд безопасности** → summary.
3. `[GUARD]` **Слайд безопасности — из фактов, не выдуманный.** Если в графе есть факт о
   противопоказании/осторожности (напр. AHA → фотосенсибилизация) — использовать его с
   `source_ref`. Если по теме нет такого факта (напр. ниацинамид хорошо переносится) — это
   слайд переносимости/применения на основе имеющихся фактов, а НЕ подделанное противопоказание.
   Никаких клинических утверждений без факта в графе.
4. `[GUARD]` Каждый клинический слайд: непустой `source_refs` (реальные fact/source ID) +
   дисклеймер «Только для специалистов». Слайд без источника не генерируется.
5. Лимиты: title ≤50, subtitle ≤100; цвет/шрифт НЕ в spec (резолв из токенов). Валидировать по
   `schemas/slide-spec.json`.

Выход: `05_content/specs/deck_vitamin_c/`, `deck_niacinamide/`, `deck_exfoliants/`.
