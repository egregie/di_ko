# YM PROSKIN — Generalize Claim-Support Gate (pre-scale)

Версия: 1.0 · Дата: 2026-06-10 · Предыдущий: `integrity-rollback-groundtruth` (принят — откат честный).
Цель: заменить хардкод-правила Rule 1–5 (специфичные под 8 аудированных фактов) на **обобщаемую**
проверку claim-support, прежде чем масштабироваться на 50 презентаций. Иначе на новой теме гейт
проваливается до 15%-keyword-пре-фильтра — геймящейся метрики.

> Свежей сессии: прочитай `00_PROJECT_STATE/` и `AGENTS.md`. Граф восстановлен (rollback принят).

Легенда: `[GUARD]` · `[ВЕРИФ]`.

---

## Проблема
Текущий гейт: 15% keyword-пре-фильтр + хардкод Rule 1–5 (RAR/RXR, tazarotene, single-step и т.д.).
Правила привязаны к конкретным 8 фактам и НЕ работают для произвольных новых тезисов. Для пайплайна
50 дек нужен общий механизм «поддерживает ли абстракт ИМЕННО этот тезис», не зависящий от темы.

## Блок A — Grounded claim-support (LLM-as-judge на живом абстракте)
1. Заменить Rule 1–5 на общую проверку: подать в judge **тезис факта** + **реально загруженный
   абстракт** (raw fetch, не пересказ). Judge возвращает: вердикт `SUPPORTED / WEAK / UNSUPPORTED`
   + **дословное предложение(я) из абстракта**, которое поддерживает тезис.
2. `[GUARD]` Заземление обязательно: если judge не может процитировать поддерживающее предложение
   ИЗ переданного абстракта — вердикт `UNSUPPORTED`. Никакого «знания из головы», только текст абстракта.
3. Калибровка вердикта: прямое подтверждение в первичном исследовании → `SUPPORTED`; поддержка в
   нарратив-обзоре/непрямая → `WEAK` (conf cap 0.80, audit_flag); нет поддержки → `UNSUPPORTED` → карантин.
4. Keyword-overlap остаётся ТОЛЬКО дешёвым пре-фильтром (отсев заведомо нерелевантного), НЕ вердиктом.
5. `[GUARD]` Judge судит по абстракту, не по PMID/названию; никакой PMID не засчитывается без
   живого fetch его абстракта (P014).

## Блок B — Переаудит 8 фактов под ОБЩИМ гейтом (валидация)
1. Прогнать `fact_0001, 0002, 0005, 0009, 0011, 0039, 0055, 0056` через новый общий judge (без
   хардкод-правил).
2. `[ВЕРИФ]` Вердикты должны воспроизвести разумный результат (ожидаемо близко к 4/2/2). Если общий
   гейт даёт ИНОЙ вердикт, чем хардкод-правило — это сигнал, что правило маскировало реальную
   картину; разобрать каждое расхождение честно (не подгонять под прошлый результат).
3. Удалить хардкод Rule 1–5 из `evidence.py` после подтверждения.

## Блок C — Регресс на обобщаемость
Добавить кейсы, которых НЕТ среди 8 «знакомых» фактов: 1 заведомо поддержанный новый тезис, 1
тезис с нерелевантным абстрактом, 1 тезис с частичной/нарратив-поддержкой (→WEAK). Гейт должен
правильно судить незнакомые тезисы, а не только зашитые концепты.

## Definition of Done
- [ ] Хардкод Rule 1–5 удалены; вердикт даёт grounded judge по живому абстракту с цитатой-доказательством.
- [ ] Без процитированного предложения из абстракта → UNSUPPORTED (заземление принудительно).
- [ ] Keyword-overlap — только пре-фильтр, не вердикт.
- [ ] 8 фактов переаудированы общим гейтом; расхождения с прошлыми вердиктами разобраны честно (не подогнаны).
- [ ] Регресс включает незнакомые тезисы (supported/irrelevant/weak) и проходит.
- [ ] `validate_graph` ок; `walkthrough.md` по строгому шаблону, честный `deltas_vs_plan`; тег `gate-generalized`.

## НЕ трогать / отложено
- `[GUARD]` Judge не выносит вердикт «из общих знаний» — только из текста переданного абстракта.
- Не подгонять переаудит 8 фактов под прошлый 4/2/2; если общий гейт честно даёт иное — принять и разобрать.
- Контент-пайплайн 50 дек (донор→гейт→матрица, начиная с Постакне/расширения ретиноидов) — СЛЕДУЮЩАЯ фаза, после обобщённого гейта.

## Открыто `[ВЕРИФ]`
- Совпадут ли вердикты общего гейта с хардкод-результатом по 8 фактам — честно сообщить расхождения.
