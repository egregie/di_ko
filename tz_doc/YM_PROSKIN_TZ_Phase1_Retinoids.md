## Блок 1 — Пилотный scope (цели сбора, НЕ утверждённые факты)
Собрать и верифицировать подграф «Ретиноиды». Ниже — **целевая таксономия** (что должно
появиться в графе). Конкретные клинические утверждения и источники заполняет пайплайн из
реального ретривала — НЕ из этого ТЗ.

Целевые сущности (entity_type: Ingredient, кроме класса):
```
CON-RETINOIDS-001     Retinoids            (Concept/Class — родитель)
ING-RETINOL-001       Retinol              aliases: vitamin A alcohol
ING-TRETINOIN-001     Tretinoin            aliases: all-trans retinoic acid, ATRA
ING-ADAPALENE-001     Adapalene
ING-TAZAROTENE-001    Tazarotene
```

Целевые связи (relationship): каждый ингредиент `is_a`/`belongs_to` → Retinoids;
`used_for` → {acne, photoaging}; `contraindicated_for` → {pregnancy, lactation};
`supports` → Study (по факту наличия исследования).

Целевые темы фактов (собрать + верифицировать, не утверждать здесь):
механизм (связывание RAR/RXR; дифференцировка кератиноцитов; подавление MMP; синтез
проколлагена), эффективность при фотостарении и акне, сравнительная активность/раздражение,
фотосенсибилизация, противопоказание при беременности/лактации.

## Блок 2 — Жёсткие ограничения `[GUARD]`
1. **EBM-only.** Источники Tier A/B (см. `sources.json`). Tier C — только контекст/темы, не
   источник клинического факта. Tier D запрещён.
2. **Доказательность.** Каждый факт: `evidence_level` A–D; **D не попадает в граф**. Клинический
   факт — ≥2 независимых источника ИЛИ 1 peer-reviewed.
3. **Никаких выдуманных источников.** `pmid`/`doi` — только из реального ретривала. Если не
   подтверждается — факт помечается `[UNVERIFIED]` и **не** пишется в граф.
4. **Копирайт.** Хранить пересказ; цитата ≤15 слов; не более одной цитаты на источник;
   провенанс обязателен.
5. **Без данных пациентов / фото «до-после».** В этом спринте — только текстовые факты.
6. **Онтология.** Один компонент = одна сущность с aliases (tretinoin = ATRA, не две).
   Контролирует A07.
