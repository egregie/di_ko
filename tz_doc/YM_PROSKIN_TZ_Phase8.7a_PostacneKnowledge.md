## Блок A — Discovery (живой PubMed-поиск)
1. Поиск кандидатов через NCBI eutils (esearch+efetch) живьём: IPv4-force + UA + throttle ≤3 req/s
   (паттерн `evidence.py`, DEC-013). Сырые выдачи — в `.tmp/` (интермедиаты, не коммитятся).
2. Целевые подтемы (клинически значимые для специалистов):
   поствоспалительная гиперпигментация (PIH) — azelaic acid, tretinoin/ретиноиды, ниацинамид,
   химические пилинги (glycolic/salicylic); атрофические рубцы постакне — microneedling,
   фракционный лазер, TCA CROSS; поствоспалительная эритема (PIE).
3. `[GUARD]` Источник↔тезис: тезис формулировать ТОЛЬКО из содержимого конкретного абстракта,
   не «по теме рядом» (урок vitamin_c/Pinnell). Никаких PMID по памяти — только из живой выдачи.

## Блок B — Кандидаты + живой гейт
1. Кандидаты `fact_0063+` в `02_processing/verify/candidates/`; источники `SRC-A054+` (tier A)
   в `sources.json` — данные (PMID, название, журнал) из живой выдачи.
2. `evidence_level` назначать по фактическому pubtype источника (A: RCT/мета/системат. обзор/
   clinical trial; B: cohort/guideline) — иначе EVIDENCE_MISMATCH.
3. Прогнать каждый кандидат: `python ops/scripts/verify_gate.py <file> --force-live`.
4. `[GUARD]` Ждать реального брака (исторически 25–33%). Браковать честно, **не добивать до
   порога мусором** (P007/P014); 0% брака = красный флаг. Собрать ~12–16 кандидатов под цель ≥8.
5. `[GUARD]` P012: недоступность NCBI или Gemini-судьи = blocked, без симуляции.

## Блок C — Онтология / граф
1. Новые сущности только те, на которые реально ссылаются прошедшие факты (P008):
   `postacne` (Condition), `post_inflammatory_hyperpigmentation` (Condition),
   `atrophic_acne_scars` (Condition), `azelaic_acid` (Ingredient/Active),
   процедуры (Procedure) по факту. Типы — строго из `ontology_v1.json`.
2. Связи (treats/is_a/related_to) + fact_ids на сущностях; `build_index.py`;
   `validate_graph.py` зелёный; `run_regression.py` без регрессий.
