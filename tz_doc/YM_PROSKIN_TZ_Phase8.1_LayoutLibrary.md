## БЛОК B — ИЗВЛЕЧЕНИЕ ГЕОМЕТРИИ (из hd_1, не из воображения)

### B1. Пер-слайдовая разметка
Для каждого слайда hd_1 извлечь фактические bounding-box'ы (в относительных долях canvas, @1280×720 как база):
- текстовые блоки → классифицировать по брендбуку: display_h1 / h2 / h3 / body / body_sm / caption.
- визуальные области → классифицировать как placeholder-типы: id_img / id_graph / id_illustration / id_logo.
- футер-хром (бренд-метка, номер слайда, источники) — зарезервированная полоса (низ ~0.08H, из brand-layer).

[GUARD] Координаты — из реального файла (измерены), помечать `[FACT]`. Где блок неоднозначен — `[UNKNOWN]`, не угадывать.

### B2. Кластеризация в layout-типы
Сгруппировать слайды hd_1 по повторяющейся композиции в **переиспользуемые layout-паттерны**. Ожидаемый пул (из брендбука Diagram Engine): Title, Big Quote/Thesis, 2-Column Split (Left Graphic / Right Text), Multi-card Grid, Analytical Chart/Table, Step-by-Step Timeline. НЕ навязывать все 6 — взять те, что реально есть в hd_1; если есть новые — задокументировать.

[GUARD] Это НЕ 12 универсальных интентов (DEC-020 предостерёг от преждевременной универсализации). Только layout'ы, подтверждённые реальным контентом донора.

### B3. Соответствие требованию 13–20
Подтвердить: набор layout'ов покрывает деку 13–20 слайдов. Если hd_1 даёт <13 уникальных композиций — это норма (layout'ы переиспользуются между слайдами); показать, как 13–20 слайдов раскладываются на N layout-типов.

## БЛОК C — LAYOUT LIBRARY v2 (формализация)

### C1. Схема каждого layout
Записать в `04_design_system/layouts_v2.json`. Для каждого layout:
```
layout_id, name, description,
canvas: {w, h},
slots: [
  { slot_id, kind (content|placeholder), block_type (display_h1|h2|h3|body|body_sm|caption | id_img|id_graph|id_illustration|id_logo),
    bounds: {x,y,w,h} (абсолют @canvas),
    relative_bounds: {x,y,w,h} (доли 0..1),
    constraints: {max_chars (для текста, из брендбука: h1≤50, callout≤100), min_size (для лого ≥18px)} }
],
footer_reserved: true, source_slide_ref (какой слайд hd_1 это породил)
```
[GUARD] Совместимо с `placeholder_contract.json` (Phase 8.0) — composite ID, bounds+relative_bounds, routing уже определены там. НЕ дублировать схему, наследовать.

### C2. 2 утверждённых каркаса
Из layout-пула собрать **ровно 2 deck-template** (`deck_template_retinoids`, `deck_template_postacne`):
- последовательность layout'ов на 13–20 слотов-слайдов.
- title=descriptor-лого, закрытие=badge (из brand-layer Phase 4.5/8.0).
- зафиксировать в `04_design_system/deck_templates.json`.
[GUARD] Шаблон НЕ содержит контента — только композиция (требование заказчика вербатим).

### C3. Рендер-доказательство (КРИТИЧНО — показать заказчице ДО наполнения)
Отрендерить **пустые каркасы** обоих шаблонов (placeholder-области показаны серыми плашками с подписью типа+ID, без контента) в PDF.
- [ВЕРИФ] НЕ литеральный текст `[Placeholder]` на слайде (старый дефект) — серая область с типом/ID-меткой.
- [ВЕРИФ] футер-хром виден, не накладывается; бренд-метка присутствует; Zero Black соблюдён.
- Показать оба пустых каркаса как реальный PDF (не локальные недоступные пути).
