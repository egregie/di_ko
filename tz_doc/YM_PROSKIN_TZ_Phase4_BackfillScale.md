# YM PROSKIN — Phase 4: Backfill + Scale Collection (Exa pilot)

Версия: 1.0 · Дата: 2026-06-07 · Предыдущий: Phase 3.5 (dual-render ПРИНЯТ).
Предусловие запуска: **визуальное ревью** `deck_retinoids_v2.pdf` и `.pptx` пользователем пройдено
(вёрстка/эстетика ок). Если есть дизайн-проблемы — сначала их фикс, потом этот спринт.
Цель: добить тонкие темы до deck-readiness (P011 ≥8 фактов) и пилотно подключить Exa, чтобы
снизить брак сбора относительно baseline 27% (DEC-010). Рендер не трогаем — он готов.

Легенда: `[GUARD]` · `[ВЕРИФ]` · `[НЕТ ДАННЫХ]`.

---

## Блок A — Пилот Exa MCP (снизить 27% брака)
1. `[GUARD]` Перед подключением: вет официального **Exa MCP** (источник, ключ, объём прав);
   подключать как MCP, не сторонний скил. Не в global вслепую; отдельный коммит.
2. Подключить Exa к A01/A02 ТОЛЬКО для одной темы бэкфилла (напр. Vitamin C).
3. Прогнать ту же тему двумя путями для сравнения: текущий поиск vs Exa-grounded сбор.
   `[GUARD]` Все факты — через живой verify_gate (`--force-live` на выборке); answer-key запрещён.
4. Зафиксировать rejection-rate каждого пути в `ops/logs/exa_pilot.md`; сравнить с baseline 27%.
5. `DEC-012`: оставить Exa или нет — по факту (брак, доля Tier-A, стоимость ~$5/1K).

## Блок B — Бэкфилл тонких тем до P011 (≥8 чистых фактов)
Темы из Batch 1, не дотянувшие до порога: **Vitamin C, Niacinamide, Exfoliants (AHA/BHA)**.
1. Для каждой: дособрать факты под живым гейтом до ≥8 чистых (supported, не weak-only).
   `[GUARD]` Не добивать мусором: если по теме реально нет ≥8 Tier-A/B — зафиксировать предел в
   `known_problems.md`, тема остаётся не-deck-ready (P011), это допустимо.
2. Per-topic rejection-rate → `ops/logs/scale_quality.md`. `[GUARD]` >30% по теме → пауза и разбор.

## Блок C — Онтология после добавлений (A07 глобально)
1. Прогнать A07 по всему графу: 0 дублей вещества между темами; новые aliases слиты
   (ascorbic acid/vit C; nicotinamide/vit B3; деривативы C → родитель Vitamin C).
2. Кросс-темные связи — только с источником+evidence (`interacts_with`, общий `used_for`).
   `[GUARD]` Без пруфа — не создавать.
