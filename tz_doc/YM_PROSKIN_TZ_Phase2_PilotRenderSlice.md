# YM PROSKIN — Phase 2: Pilot Render Slice (Retinoids Deck)

Версия: 1.0 · Дата: 2026-06-07 · Предыдущий: Phase 1.5 (аудит принят).
Цель: достроить вторую половину вертикального среза — `граф → slide-spec → PDF` — на 12
аудированных фактах. Это **тонкий срез для проверки рендер-пайплайна**, НЕ полная
дизайн-система и НЕ масштабирование сбора.

Легенда: `[GUARD]` жёсткое ограничение · `[ВЕРИФ]` · `[НЕТ ДАННЫХ]`.

## Блок B — Slide-spec из графа (A09 architect)
1. Deck-plan пилотной колоды (6–8 слайдов), темы — только из чистого графа:
   - cover (YM PROSKIN · Ретиноиды)
   - что такое ретиноиды (класс Retinoids)
   - механизм действия (RAR/RXR, дифференцировка, MMP/коллаген)
   - ключевые активы — сравнение (three_cards: retinol/tretinoin/adapalene)
   - эффективность (акне/фотостарение)
   - **противопоказание — беременность/лактация (contraindication_alert)** — обязательный слайд
   - summary
2. A09 берёт факты ТОЛЬКО через `retriever.py` из очищенного графа.
   `[GUARD]` Исключить из графа для рендера: quarantined, `needs_manual`. Для **headline-claims**
   использовать только clean-факты (не weak); weak — допускаются лишь как вторичные, с осторожностью.
3. Каждый `slide-spec.json` валиден по `schemas/slide-spec.json`: title≤50, subtitle≤100,
   БЕЗ цвета/шрифта; `source_refs[]` = реальные entity/fact/source ID из графа.
4. `[GUARD]` Каждый клинический слайд несёт `disclaimers` («Только для специалистов») и
   непустой `source_refs`. Слайд без источника не генерируется.

Выход: `05_content/specs/deck_retinoids/*.json`.

## Блок C — Рендер (HTML/CSS → PDF, канон)
1. `deck-render` (skill): `slide-spec` → HTML/CSS, цвета = CSS-переменные из `design-tokens.json`
   дословно → экспорт в PDF. python-pptx — отложен (не в этом срезе).
2. Изображения: для пилота только плейсхолдеры/схемы-заглушки. `[GUARD]` Никаких фото пациентов.
3. Картинки кадрировать под слот (cover), не растягивать.
Выход: `06_render/out/deck_retinoids.pdf` (+ промежуточный HTML).
