# YM PROSKIN — Phase 5.1: Deck Integrity Hardening

Версия: 1.0 · Дата: 2026-06-08 · Предыдущий: Phase 5 (принят, тег `phase-5-topic-decks`).
Причина: жёсткая pregnancy-проверка в `qa_deck.py` применяется ко всем темам и заставила вписать
в деки Vit C / Niacinamide / Exfoliants утверждения о безопасности при беременности, сославшись
на факты, которые про беременность ничего не говорят. QA это пропустил, т.к. проверяет НАЛИЧИЕ
`source_ref`, а не то, что источник подтверждает тезис. Цель — закрыть зазор слайд↔факт.

> Свежей сессии: сначала прочитай `00_PROJECT_STATE/` и `AGENTS.md` (P001–P017), потом исполняй.

Легенда: `[GUARD]` · `[ВЕРИФ]` · `[НЕТ ДАННЫХ]`.

---

## Блок A — Topic-aware safety-проверка в `qa_deck.py`
1. `[GUARD]` Снять жёсткое требование «беременность/лактация на каждой деке».
2. Новое правило: каждая дека ОБЯЗАНА иметь **слайд безопасности**; pregnancy/lactation-контент
   требуется ТОЛЬКО если у темы есть факт в графе про беременность (или тема входит в явный
   config-список контраиндикаций, напр. retinoids). Иначе слайд безопасности = переносимость
   из имеющихся фактов.
3. Конфиг тем-контраиндикаций вынести в данные (не хардкод по строке), чтобы не зашивать «беременность» в grep.

Acceptance: ретиноиды по-прежнему требуют pregnancy-слайд; Vit C/Niacinamide/Exfoliants — нет;
QA не вынуждает выдумывать.

---

## Блок B — Проверка связки слайд↔факт (ядро спринта)
`[GUARD]` Наличие `source_ref` НЕ достаточно — источник должен реально поддерживать тезис.
1. Добавить в `qa_deck.py` (или новый `qa_claims.py`) проверку: для каждого клинического слайда
   каждое клиническое утверждение должно следовать из `statement` хотя бы одного процитированного
   факта.
2. Минимум на этот спринт: флагать клинические утверждения (особенно безопасность: беременность,
   противопоказание, «безопасен/эффективен при X»), которых НЕТ в тексте ни одного
   процитированного факта. Такой слайд → BLOCK.
3. Это закрывает зазор: verify_gate гарантирует факт↔источник на сборе, теперь QA гарантирует
   слайд↔факт на рендере.

Acceptance: проверка ловит claim, отсутствующий в процитированных фактах (тест на текущих
pregnancy-слайдах должен сработать как FAIL до фикса Блока C).

---

## Блок C — Аудит и фикс трёх дек
1. Просмотреть слайды безопасности Vit C / Niacinamide / Exfoliants. Каждое pregnancy/клиническое
   утверждение без поддерживающего факта:
   - либо собрать реальный факт о профиле беременности ингредиента через живой `verify_gate`
     (если Tier-A есть) и сослаться на него;
   - либо переписать слайд как переносимость из имеющихся фактов и **убрать неподтверждённое
     утверждение** (P007: лучше меньше, но честно).
   `[GUARD]` Никаких клинических утверждений без факта в графе. Не «клинически правдоподобно» — а подтверждено.
2. Перерендерить три деки (PDF+PPTX) и прогнать обновлённый QA (Блоки A+B) — PASS.

Acceptance: 0 неподтверждённых клинических утверждений на трёх деках; обе проверки (A+B) зелёные.
