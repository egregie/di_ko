## Блок A — Диагностика `fact_0030` / `fact_0046`
1. Открыть JSON обоих фактов; прочитать поле `sources`/PMID и `statement`.
2. Определить случай:
   - **(A1) Факт ссылается на PMID 11207686** для тезиса про мРНК коллагена → это **рассинхрон
     факт↔источник**: источник не подтверждает тезис (verify_gate должен был отсечь). 
   - **(A2) Факт ссылается на корректный источник**, а Pinnell проставлен только в подписи схемы →
     это **баг подписи диаграммы** (диаграмма подтянула не тот PMID независимо от факта).
3. Действие:
   - При A1: `[GUARD]` НЕ подставлять «правдоподобный» PMID вручную. Найти и **живым гейтом**
     верифицировать корректный источник тезиса мРНК-коллагена (кандидат — Nusgens BV et al., 2001,
     J Invest Dermatol; PMID подтвердить через PubMed live). Источник подтверждает тезис → заменить;
     не находится → **в карантин** (P007/P009), не оставлять с битым источником.
   - При A2: исправить источник в данных схемы (`gen_diagrams.py`/spec), чтобы подпись совпадала с
     реальным источником факта; факт не трогать.

## Блок B — Sweep источник↔тезис по активному графу
1. Прогнать проверку claim-support по ВСЕМ активным фактам (не только vitamin_c): для каждого —
   подтверждает ли процитированный источник ИМЕННО этот тезис, а не «по теме рядом».
2. Любой найденный рассинхрон — зафиксировать списком и обработать как в Блоке A (live-реверификация
   либо карантин). `[GUARD]` Ожидать, что мог быть не один; сообщить честный список, не «всё чисто» по умолчанию.

## Блок C — Усилить claim-support в гейте (если sweep покажет послабление)
Если рассинхроны нашлись — значит `verify_gate` где-то матчит по теме/ключевым словам, а не по
тезису. Ужесточить: источник засчитывается, только если абстракт поддерживает конкретное
утверждение факта. (Та же дисциплина, что слайд↔факт P018 и источник↔тезис из Phase 6.)
